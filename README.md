# 🐄 CowSimulator

一款类似数独、数织的逻辑推理谜题游戏。玩家通过逻辑推理，找出隐藏在彩色格子矩阵中的所有「标定方块」。

---

## 📁 文件结构

```
CowSimulator/
├── index.html       # 主页（入口）
├── Editor.html      # 关卡编辑器
├── Simulator.html   # 关卡模拟（游玩页）
├── style.css        # 共享样式（深色主题）
├── README.md        # 本说明文档


---

## 🎮 游戏规则

### 目标
找出矩阵中所有隐藏的「标定方块」。

### 标定方块的规则
1. **颜色唯一性**：每种颜色区域有且仅有一个标定方块。
2. **行列唯一性**：矩阵的每行、每列有且仅有一个标定方块。
3. **不相邻**：任意两个标定方块不能相邻，包括斜角（即每个标定方块周围的 8 个格子中均不能有其他标定方块）。
4. **格子唯一**：每个格子最多只有一个标定方块。

## 🖥️ 页面说明

### 主页（index.html）
入口页面，提供两个功能入口：
- **关卡编辑器**：前往 `Editor.html`
- **关卡模拟**：前往 `Simulator.html`

---

### 关卡编辑器（Editor.html）

用于设计和保存关卡。

| 功能 | 操作说明 |
|------|---------|
| 新建关卡 | 输入矩阵大小（2×2 ~ 20×20），创建空白网格 |
| 小格赋色 | 激活后弹出可拖动颜色面板，鼠标按住拖动批量涂色；再次点击已有颜色格可清除 |
| 设定标定方块 | 点击格子设置标定方块，自动进行四项合法性校验；再次点击取消 |
| 保存关卡 | 将关卡导出为 `.cow` 文件，同时自动运行解密并将答案写入文件 |
| 读取关卡 | 从 `.cow` 文件恢复关卡数据 |
| 清空颜色 | 清除所有格子颜色（同时清除标定方块） |
| 清空标定 | 仅清除所有标定方块，保留颜色 |
| 🤖 自动解密 | 运行自动求解算法，将结果显示在编辑器中 |
| ◀ 返回主页 | 返回 `index.html` |

#### 颜色面板
- 支持 15 种颜色 + 1 种「清除」选项
- 面板可通过鼠标拖动移位
- 点击色块切换当前绘制颜色

---

### 关卡模拟（Simulator.html）

用于游玩关卡。

| 功能 | 操作说明 |
|------|---------|
| 读取关卡 | 加载 `.cow` 文件，正确答案自动隐藏 |
| 刷新关卡状态 | 清空所有玩家标记，回到初始状态 |
| 确认答案 | 与文件中的答案对比，全对则胜利，有误则提示错误数量 |
| ◀ 返回主页 | 返回 `index.html` |

#### 游玩操作
| 操作 | 效果 |
|------|------|
| **双击**格子 | 标记为「🐄 标定方块」；再次双击取消 |
| **单击**格子 | 标记为「✕ 排除」；再次单击取消 |
| **拖动**鼠标 | 批量标记/取消「✕ 排除」 |

顶部徽章实时显示剩余需标记的标定方块数量。

## 💾 .cow 文件格式

`.cow` 文件为 JSON 格式，结构如下：

```json
{
  "version": 1,
  "n": 8,
  "grid": [
    [1, 1, 2, 2, 3, 3, 4, 4],
    ...
  ],
  "markers": ["0,3", "2,7", "4,1", "6,5"],
  "solution": ["0,3", "2,7", "4,1", "6,5"]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `version` | number | 文件格式版本号，当前为 1 |
| `n` | number | 矩阵大小（n×n） |
| `grid` | number[][] | n×n 颜色索引矩阵，0 为空白，1~15 为颜色编号 |
| `markers` | string[] | 编辑器中手动设置的标定方块位置，格式为 `"行,列"`（0-indexed） |
| `solution` | string[] \| null | 自动解密算法计算出的答案；若关卡无解则为 `null` |

## 🤖 自动解密算法

自动解密模块位于 `Editor.html` 的 `autoSolve()` 函数，采用**约束传播 + 回溯搜索（AC-3 风格）**。

### 算法流程

```
第一层：候选域初始化
  └── 对每种颜色，收集网格中所有属于该颜色的格子作为初始候选集

第二层：约束传播（循环直到稳定）
  ├── 规则 A（单候选确定）
  │     若某颜色只剩 1 个候选格 → 确认放置
  │     删除该格所在行、列及 8-邻域中其他颜色的候选
  ├── 规则 B（行独占性）
  │     若某行只有 1 种颜色有候选格 → 该颜色必须放在此行
  │     删除该颜色在其他行的候选
  └── 规则 C（列独占性）
        若某列只有 1 种颜色有候选格 → 该颜色必须放在此列
        删除该颜色在其他列的候选

第三层：回溯搜索（MRV 启发式）
  └── 选择候选数最少的颜色优先枚举（最小剩余值策略）
      对每个候选位置创建分支 → 递归 → 矛盾则回溯
```

### 复杂度
- 约束传播通常可在多项式时间内将大部分关卡直接求解
- 仅当传播无法继续时才进入回溯搜索
- MRV 启发大幅减少搜索树节点数，实际性能良好

## 🚀 运行方式

直接在浏览器中打开 `index.html` 即可，无需服务器或构建工具。

建议使用现代浏览器（Chrome / Firefox / Edge / Safari）以获得最佳体验。

## 📌 开发说明

- 所有页面为纯 HTML + CSS + JavaScript，无外部依赖
- 公共样式集中在 `style.css`，各页面内联少量页面特有样式
- `.cow` 文件本质为 JSON，可用文本编辑器直接查看和修改
- 自动解密接口为独立函数 `autoSolve(n, grid, markers)`，输入输出格式明确，便于替换算法实现
